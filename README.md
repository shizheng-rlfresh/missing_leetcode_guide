# LeetCode Notes: Contract → Hypothesis → Invariant

Minimal notes for LeetCode-style coding interviews, following one workflow:

**Contract → Hypothesis → Invariant**

- **Contract:** input/output + edge cases + constraints  
- **Hypothesis:** candidate algorithm family (e.g., two pointers / binary search)  
- **Invariant:** correctness certificate maintained throughout the algorithm
